# [상] 10차시 구현 과정 기록표 및 점검표

## 1. 구현 과정 기록표

우리 팀은 **이동약자(고령자, 관절염 환자 등)가 서점에서 겪는 보행 장애 환경**을 주제로 시뮬레이터를 구현하였습니다. 아래는 AI(바이브 코딩)와의 협업을 통해 3D 환경으로 장애물을 구체화하고 개선해 나간 기록입니다.

| 항목 | 내용 |
| :--- | :--- |
| **① 구현하려고 한 장애물 5개** | 1. **고중량 수동 통유리문**: 일반인은 쉽게 여는 문이지만 근력 저하자가 양손으로 힘겹게 밀어야 하는 저항력을 모사<br>2. **좁은 서가 구간**: 높은 서가들이 밀집하여 휠체어나 보행 보조기 이용 시 좌우 방향 전환과 회피가 어려운 좁은 동선<br>3. **혼잡한 시민 동선 (NPC)**: 주변을 빠르게 지나다니는 비장애인들과의 보행 속도 차이 및 충돌 스트레스<br>4. **높은 의자 (바 체어)**: 도서 검색대 옆에 배치된 높은 의자로, 관절을 지지하지 못해 짧은 휴식조차 고통이 되는 상황 모사<br>5. **핸드레일 없는 계단**: 경사가 급하지만 신체를 지지할 손잡이가 없어 낙상 위험이 극대화되는 계단 구간 |
| **② 특수 조건 반영** | - **'관절 내구도(스태미나)'** 수치를 도입하여 플레이어의 물리적 한계를 정량화함.<br>- 특정 장애물 및 위험 구역에 진입할 때마다 관절 내구도가 실시간으로 감쇠함.<br>- 내구도가 20% 미만으로 떨어질 경우 '관절 손상 경고'가 뜨며, 최종 목적지에 도달해도 미션을 재도전해야 하는 패널티 조건을 설정함. |
| **③ 이동 경로 및 목적지** | - **이동 경로**: [출발] 서점 입구 앞 $\rightarrow$ 고중량 수동 유리문 통과 $\rightarrow$ 좁은 서가 동선 우회 $\rightarrow$ 시민 NPC 회피 $\rightarrow$ [목적지] 핸드레일 없는 계단 앞<br>- 바닥에 **파란색 유도선(가이드 라인)**을 배치하여 이동 경로를 시각적으로 안내함. |
| **④ 충돌 처리 구현** | - 플레이어의 바운딩 박스(AABB)와 벽/서가 오브젝트의 충돌을 감지하여 벽을 뚫고 나가지 못하도록 차단함(`collides` 함수).<br>- 이동 경로 내 움직이는 10명의 시민 NPC들과 충돌하거나 너무 밀접해질 경우(거리 0.85m 이내), 감속 패널티(0.42 m/s)와 함께 스태미나가 지속적으로 깎이도록 설계함. |
| **⑤ 이벤트 트리거** | - **수동문 영역 진입**: `doorTask` UI가 화면 중앙 하단에 팝업되며 스페이스바를 연타해야 문이 열리는 게이지 트리거 작동.<br>- **계단 영역 진입**: 화면 테두리가 붉게 타오르는 '통증 비네트(severe vignette)' 효과와 카메라 흔들림(Camera Shake)이 발생하여 시각적 위기감 극대화. |

### AI 협업 및 프롬프트 기록

*   **사용한 프롬프트 (히스토리)**
    1.  *(기본 환경 구축)* `"Three.js와 Cannon.js를 사용해서 이동약자가 서점 내에서 겪는 장애물을 체험할 수 있는 3D 시뮬레이터를 만들어줘. 기본 속도와 스태미나 게이지가 있어야 해."`
    2.  *(1차 수정)* `"입구의 수동유리문을 열 때 단순히 문을 지나가는 게 아니라, 스페이스바를 연타해야만 열리는 무거운 느낌과 관절 부담(스태미나 감소)을 추가해줘."`
    3.  *(2차 수정)* `"좁은 서가 사이를 돌아다니는 일반 시민(NPC)들을 추가하고, 그들과 부딪히거나 가까워지면 당황하거나 속도가 느려지고 관절 부담이 가중되는 연출을 넣어줘."`
    4.  *(3차 수정)* `"마지막 목적지인 계단에 핸드레일이 없는 위험 상황을 묘사하고, 계단 앞에 도달했을 때 시각적 피드백(화면이 붉어지는 비네트 효과)과 강한 경고 메시지를 보여줘."`

*   **1차 수정 후 결과**
    *   수동문에 물리 힌지 조인트(`CANNON.HingeConstraint`)와 플레이어 입력 힘(`doorForce`)이 결합되어, 스페이스바를 다다닥 눌러야만 문이 서서히 밀려 열리는 묵직한 수동문 조작감이 성공적으로 구현됨.
*   **2차 수정 후 결과**
    *   `THREE.InstancedMesh`를 통해 최적화된 10명의 시민 NPC가 좌우로 왕복하는 동선이 생성됨. NPC 근처에 가면 스태미나가 급격히 떨어지며 보행 방해를 체감할 수 있게 됨.
*   **3차 수정 후 결과**
    *   계단 양옆에 핸드레일이 유실되었음을 알리는 빨간색 위험 경고 마커가 추가되었고, 플레이어가 계단 영역에 발을 들이는 순간 화면에 강렬한 적색 비네트 필터와 함께 카메라 진동이 연출되어 위험성이 시각적으로 뚜렷해짐.

## 2. 구현 과정 점검표

| 질문 | 작성 내용 |
| :--- | :--- |
| **가장 어려웠던 구현은 무엇인가?** | 물리 엔진(Cannon.js)의 힌지 제약조건을 사용해 유리문을 회전시키면서, 동시에 웹 브라우저 이벤트(Space 연타)와 스태미나 수치를 프레임 단위로 유기적으로 연동하는 부분이 가장 까다로웠습니다. 단순한 값의 변화가 아니라 사용자가 '무겁다'는 느낌을 직관적으로 받도록 감쇠비(damping)를 조정하는 세밀한 튜닝이 필요했습니다. |
| **원하는 결과가 바로 나왔는가?** | 아니오. 초기 프롬프트 입력 시에는 유리문이 종이처럼 가볍게 날아가거나 스페이스바 한 번에 완전히 활짝 열려버렸습니다. 또한 NPC들이 벽을 뚫고 나가거나 플레이어와 부딪혔을 때 끼임 현상이 일어나 조작이 멈추는 에러가 있었습니다. |
| **결과가 다르게 나왔을 때 어떻게 수정했는가?** | AI에게 물리 충돌의 범위를 직접 계산하는 복잡한 물리 계산 대신, 거리 기반 감지(Distance checks)와 경고 상태 변수를 활용해 가벼운 이벤트 기반 연출로 전환하라고 요청했습니다. 유리문 또한 물리 법칙을 간소화하고 `THREE.MathUtils.damp`를 통해 프레임 속도 독립적인 부드러운 회전 보간법을 적용하도록 코드를 수정했습니다. |
| **AI에게 설명할 때 중요했던 점은 무엇인가?** | "불편하게 만들어줘", "문이 무겁게 열리게 해줘" 같은 주관적 감상어 대신, `"Space를 입력할 때마다 doorForce 수치를 0.13씩 더하고, 프레임마다 0.09씩 감쇠하며, 스태미나를 1.8씩 감소시킨다"`처럼 **개발적인 변수명과 정량화된 수치 조건**을 구체적으로 지시하는 것이 원하는 아키텍처를 빠르게 뽑아내는 데 핵심적이었습니다. |
| **다음에 구현한다면 추가하고 싶은 기능은 무엇인가?** | 현재는 웹 화면과 키보드 연타로만 체험하지만, 실제 아두이노(Arduino)와 압력 센서를 연동하여 손에 쥐는 손악력 기기나 스마트폰 가속도 센서를 흔들어 실제로 몸에 힘을 주어야 문을 밀 수 있는 **실감형 IoT 하드웨어 연동 기능**을 확장하고 싶습니다. |
